# Contributing to Bulletproof

Thanks for your interest in making Bulletproof better!

## Ways to Contribute

- **Report bugs** — [open an issue](https://github.com/artemiimillier/bulletproof/issues)
- **Share examples** — add a before/after use case to `examples/`
- **Improve the workflow** — suggest stage improvements or new checks
- **Fix typos and docs** — every improvement counts
- **Share the project** — star the repo, tell your friends, post about it

## How to Submit a Change

1. Fork the repository
2. Create a branch: `git checkout -b my-improvement`
3. Make your changes
4. Commit with a clear message: `git commit -m "add: new example for React refactoring"`
5. Push: `git push origin my-improvement`
6. Open a Pull Request — describe what you changed and why

## Guidelines

- **Keep SKILL.md structure intact** — it's the core of the skill. If you want to restructure it, open an issue for discussion first.
- **Templates should stay minimal** — they're starting points, not comprehensive documents.
- **Examples should be realistic** — show real problems that developers actually face with AI agents.
- **One change per PR** — easier to review and merge.

## What We're Looking For

- Real-world examples of Bulletproof catching bugs that would have shipped
- Workflow improvements backed by experience (not theory)
- Compatibility reports with different AI coding tools
- Translations of README to other languages

## Response Time

We aim to respond to issues and PRs within 7 days.
